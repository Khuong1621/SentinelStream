# Architecture Documentation

## Directory Structure
- `/backend`: Go or .NET source code.
- `/frontend`: .NET source code.
- `/docs`: Project documentation.

## Coding Standards
Refer to [docs/CONVENTIONS.md](CONVENTIONS.md) for detailed coding standards and naming conventions.

- **Backend:** Follow Go or .NET (C#) conventions.
- **Frontend:** Follow .NET (C#) and Tailwind CSS standards.
- **Styling:** Use Tailwind CSS for all UI styling.
- **Files:** Follow a clear and organized folder structure.
